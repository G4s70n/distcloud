ARQUITECTURAS DISTRIBUIDAS

Gastó-n Nieto

De Cluster a Cloud: cómo cooperan miles de máquinas para resolver problemas complejos

El Poder de la Colaboración Computacional

Una arquitectura distribuida es un sistema cuyos componentes están ubicados en diferentes máquinas conectadas en red, que se comunican y coordinan sus acciones mediante el paso de mensajes.

Objetivo principal: Mejorar el rendimiento, la escalabilidad, la fiabilidad y compartir recursos de forma eficiente.

CLUSTER COMPUTING: Fuerza Unificada

¿Qué es?

Un conjunto de computadoras (nodos) homogéneas, conectadas por una red de alta velocidad (generalmente LAN), que trabajan juntas como un único y potente sistema. A menudo, los nodos comparten almacenamiento y son gestionados por un software especializado.

Características Clave:

  • Acoplamiento Fuerte: Nodos estrechamente interconectados.
  • Homogeneidad: Nodos con hardware y software similar o idéntico.
  • Gestión Centralizada: Se administra como una sola entidad.
  • Imagen de Sistema Único (SSI): A veces, aparece como una única máquina.
  • Alta Disponibilidad (HA): Tolerancia a fallos.
  • Equilibrio de Carga: Distribución de tareas.

Ventajas:

  • Alto rendimiento para cómputo paralelo.
  • Buena relación precio/rendimiento.
  • Alta disponibilidad y tolerancia a fallos.

Desventajas:

  • Escalabilidad limitada por LAN.
  • Menor flexibilidad geográfica.
  • Administración puede ser compleja.

Casos de Uso Típicos:

  • Servidores web de alto tráfico, bases de datos HPC, renderizado.

GRID COMPUTING: Colaboración a Gran Escala

¿Qué es?

Una infraestructura distribuida que agrupa recursos computacionales heterogéneos y geográficamente dispersos, de diferentes dominios administrativos, para resolver problemas a gran escala. Se enfoca en compartir recursos entre 'Organizaciones Virtuales' (VOs).

Características Clave:

  • Acoplamiento Débil: Recursos conectados por internet/WAN.
  • Heterogeneidad: Diversidad de hardware y software.
  • Distribución Geográfica: Múltiples ubicaciones.
  • Gestión Descentralizada: Cada organización controla sus recursos.
  • Uso de Middleware: Software para gestión, seguridad y acceso.

Ventajas:

  • Acceso a enorme poder de cómputo.
  • Utilización de recursos infrautilizados.
  • Colaboración interinstitucional.

Desventajas:

  • Complejidad de gestión y seguridad.
  • Ancho de banda puede ser cuello de botella.
  • Fiabilidad de recursos no garantizada.

Casos de Uso Típicos:

  • Proyectos científicos (SETI@home, LHC), análisis de datos masivos, simulaciones.

CLOUD COMPUTING: Flexibilidad Bajo Demanda

¿Qué es?

Un modelo que permite el acceso ubicuo, conveniente y bajo demanda a un conjunto compartido de recursos computacionales configurables que pueden ser rápidamente aprovisionados y liberados.

Características Esenciales (NIST):

  • Autoservicio Bajo Demanda
  • Acceso Amplio a la Red
  • Agrupación de Recursos
  • Elasticidad Rápida
  • Servicio Medido (Pago por Uso)

Modelos de Servicio:

  • IaaS: Infraestructura como Servicio (VMs, almacenamiento).
  • PaaS: Plataforma como Servicio (desarrollo y gestión de apps).
  • SaaS: Software como Servicio (apps listas para usar).

Modelos de Despliegue:

  • Nube Pública, Privada, Híbrida, Comunitaria.

Ventajas:

  • Coste inicial reducido, alta escalabilidad, accesibilidad global.

Desventajas:

  • Dependencia del proveedor, preocupaciones de seguridad/privacidad.

Casos de Uso Típicos:

  • Alojamiento web/apps, almacenamiento, Big Data, desarrollo, IoT.

Escalabilidad Limitada por LAN Muy alta (global) Muy alta (bajo demanda)
Administración Centralizada, compleja Descentralizada, muy compleja Gestionada por proveedor/usuario
Propietario Única organización Múltiples organizaciones Proveedor / Organización

Virtualización Opcional Opcional Fundamental

El Futuro es Distribuido y Conectado

Cluster, Grid y Cloud Computing representan diferentes enfoques para la computación distribuida, cada uno con sus fortalezas y casos de uso ideales. A menudo, estas arquitecturas no son mutuamente excluyentes y pueden combinarse.

La tendencia es hacia una mayor abstracción, automatización y servicios bajo demanda, donde el Cloud Computing está liderando la transformación digital.
